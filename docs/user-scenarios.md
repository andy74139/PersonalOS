# User Scenarios

Personal OS is designed around real-life workflows rather than isolated features.

Each scenario should describe:

- Situation
- User Goal
- Current Pain
- Personal OS Experience
- Future Vision

---

# Scenario 1 — Morning Planning

**Situation**

The user just woke up.

**User Goal**

Understand what matters today in less than one minute.

**Current Pain**

- Calendar is in one app.
- Todos are in another app.
- Notes are somewhere else.
- The user spends mental energy deciding where to start.

**Personal OS**

Morning Brief:

- Today's focus
- Today's meetings
- Important todos
- Goal progress
- One recommendation
- One encouragement

**Future Vision**

The AI proactively identifies the most meaningful task instead of simply listing everything.

---

# Scenario 2 — Quick Capture

**Situation**

The user suddenly remembers something.

**Examples**

"Buy milk tomorrow."

"Prepare SRM tonight."

"Reply to Sylvia."

**User Goal**

Capture the thought immediately.

**Current Pain**

Typing takes too long.

Switching apps interrupts thinking.

**Personal OS**

The user simply types or speaks naturally.

The AI extracts:

- title
- due date
- priority
- life area

The todo appears instantly.

**Future Vision**

The user never needs to think about formatting.

# Scenario 4 — Meeting Notes

**Personal OS**

- Key points

---

# Design Principle

Every feature should improve at least one user scenario.

If a feature does not improve any scenario, reconsider whether it should exist.

Personal OS should always optimize for reducing cognitive load and helping users build momentum toward the life they want.
